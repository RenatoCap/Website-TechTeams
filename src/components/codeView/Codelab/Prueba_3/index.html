Inicio de Google Colab

Para utilizar Google Colab y ejecutar código de python en el navegador web acceda al siguiente enlace. Al ingresar verá una ventana emergente que nos ofrece muchas funciones.

Esta ventana emergente nos proporciona opciones para crear un nuevo cuaderno, cargar uno que hayamos realizado con anterioridad o seleccionar diferentes fuentes como Github, Google Drive o computadora local.

Podemos cargar un notebook que hayamos hecho en nuestro computador local desde la pestaña subir, en esta encontraremos un botón que nos permite seleccionar el archivo que queremos subir. Hay que tener en cuenta que el archivo tiene que ser un Jupyter Notebook (.ipynb)

También tenemos la opción de cargar nuestros notebooks desde Github, para esto necesitamos la URL de su proyecto, de la organización o usuario. Para ésto vamos a ir a la pestaña de GitHub e ingresamos el siguiente URL 

https://github.com/GDSC-UNI/Introduccion_a_Google_Colab/tree/main

y le daremos a buscar. La URL que hemos ingresado pertenece al repositorio de Google Colab de la página del GDSC UNI. Aquí abriremos el notebook con el nombre "Introducción a Google Colab".

Google Colab nos proporciona la opción de leer nuestros documentos y archivos que tenemos en nuestro Drive. Para lograr esto, primero debemos de importar el paquete que nos permitirá montar nuestro notebook en nuestro drive.

from google.colab import drive
drive.mount("/content/drive")

Al ejecutar esta celda Google nos pedirá que ingresemos nuestro código de verificación, para esto ingresamos al link que nos proporciona.

Una vez que ingrese su código de autorización ya tendrá montado Google Drive. Ahora puede leer cualquiera archivo que tengas en su drive como por ejemplo archivos CSV, módulos de python, imágenes, archivos .txt, etc.

Podemos verificar esto desde la pestaña de archivos

Podemos escribir comandos que nos permiten realizar operaciones de manera más rápida en Google Colab, para hacer uso de estos comandos debemos de utilizar el prefijo %, para conocer los magic words podemos ejecutar el siguiente comando en la celda.

%lsmagic

Anteriormente enlazamos nuestro drive con Google Colab y lo montamos en nuestra unidad. Generalmente nosotros nos encontraremos trabajando con una carpeta en la que se encontrará todo nuestros proyectos. Antes de realizar la creación de estos proyectos vamos a movernos entre las carpetas de nuestro drive, para esto primero veremos en dónde estamos y hacia donde nos podemos mover.

!ls

Cabe recalcar que el resultado dependerá de su cuenta de drive pero lo que tendremos todo en común es la carpeta "drive" a la cual nos moveremos.

%cd drive
!ls

Al listar esta carpeta tendremos otras dos carpetas, MyDrive y Sharedrives. Ahora entraremos a la carpeta MyDrive y aquí crearemos una carpeta con el nombre de "Codelab_Google_Colab" y nos moveremos a esta carpeta.

%cd MyDrive
%mkdir "Codelab_Google_Colab"
%cd "Codelab_Google_Colab"

Ahora crearemos un archivo de texto que contendrá las librerías de python que tiene Google Colab por defecto, para esto haremos un redireccionamiento de la salida del comando pip freeze hacia un el archivo requirements.txt

!pip freeze > requirements.txt

Para crear un archivo tenemos que usar el comando touch seguido de cómo queremos que se llame el archivo con su respectivo formato, por ejemplo: !touch requirements.txt

Google Colab nos da la facilidad de cargar datos de Kaggle en Colab de manera directa, para poder realizar este procedimiento es necesario tener cuenta en Kaggle ya que se requiere un token de la API de Kaggle para realizar la importación de datos. A continuación realizaremos paso a paso lo que debe de realizar para poder tener el proceso de manera satisfactoria.

Ingrese a su cuenta de Kaggle

En caso de no tener cuenta, puede crearse una ingresando al siguiente link. Una vez que tenga vaya a "Account"

Cree un nuevo Token

Dentro Account, encontrará una sección llamada API donde podrá crear un nuevo Token. Dele click a "Create New API Token". Haga clic en "Expire API Token" para eliminar el token anterior si es necesario.

Una vez que le dio click, se descarga un archivo llamado kaggle.json, el cúal contiene el nombre de usuario y la clave.

Configuración de nuestro Colab

En el notebook que hemos venido trabajando, instale el paquete de Kaggle 

! pip install -q kaggle

Después importe el siguiente paquete que le permitirá cargar archivos a su drive. 

from google.colab import files
files.upload()

Al ejecutar este código, debemos subir el archivo kaggle.json que descargamos anteriormente.

Una vez que hemos subido nuestro archivo, creamos una carpeta oculta en el sistema con el nombre .kaggle 

! mkdir ~/.kaggle

Copiamos nuestro archivo json dentro de esta carpeta

! cp kaggle.json ~/.kaggle/

Y cambiamos los permisos para permitir la lectura de nuestro archivo

!chmod 600 ~/.kaggle/kaggle.json

Haciendo la prueba de la conexión

Para comprobar que todos los pasos se han realizado de manera correcta, utilizaremos el comando list para listar algunos datasets que se encuentran disponibles 

! kaggle datasets list

Cargando datos de Kaggle

Para descargar un archivo de Kaggle debemos de ir a la página principal del dataset que queremos descargar y copiaremos el comando que nos proporcionan. 

!kaggle competitions download -c titanic

Como último paso organizaremos nuestro archivos, para esto crearemos una carpeta, con el nombre "in" el cual contendrá los archivos "gender_submission.csv", "test.csv" y "train.csv", además eliminaremos el archivo kaggle.json ya que no lo necesitamos en nuestra carpeta de trabajo. 

!mkdir "in"
!mv gender_submission.csv test.csv train.csv "in"
!rm kaggle.json
!ls

Una de las razones por las que usamos Google Colab es porque nos proporciona dos aceleradores de hardware, GPU (Unidad de procesamiento gráfico) y TPU (Unidad de procesamiento de tensores).

Configuración del GPU

Google Colab nos ofrece un acelerador de hardware gratuito GPU, el cual es requerido en Machine Learning y Deep Learning para procesar simultáneamente varios cálculos. Las GPU nos brindan un rendimiento sobresaliente a través del paralelismo y puede lanzar millones de subprocesos en una llamada.

Para configurar el GPU vaya a entorno de ejecución → cambiar tipo de entorno de ejecución y seleccione GPU en la ventana emergente.

Podemos comprobar los detalles sobre nuestra GPU en Colab utilizando el siguiente código en una celda. 

import tensorflow as tf
from tensorflow.python.client import device_lib
tf.test.gpu_device_name()

También podemos analizar los detalles del hardware utilizado para la GPU

device_lib.list_local_devices()

Configuración del TPU

La unidad de procesamiento de tensor (TPU) es utilizado para la aceleración de un gráfico de Tensorflow. Este acelerador de hardware es un circuito integrado de especificación de aplicación (ASIC) del acelerador de IA especialmente diseñado para las redes neuronales.

Para configurar el TPU vaya a entorno de ejecución → cambiar tipo de entorno de ejecución y seleccione TPU en la ventana emergente.

Al igual que con el GPU podemos comprobar el funcionamiento de nuestro TPU mediante la ejecución del siguiente código en una celda.

import tensorflow as tf
try:
    tpu = tf.distribute.cluster_resolver.TPUClusterResolver()  
    print('Running on TPU ', tpu.cluster_spec().as_dict()['worker'])
except ValueError:
    print('Exception')
